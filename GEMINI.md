# Project Context: Objectified AI

## Project Overview
This is a web application built with [Next.js](https://nextjs.org/) (App Router) and [Chakra UI](https://www.chakra-ui.com/). The project serves as a website for "Objectified AI", a thesis project. It implements a modern, responsive design using Chakra UI's component library and custom typography.

## Tech Stack
- **Framework:** Next.js 16 (App Router)
- **UI Library:** Chakra UI v3 (`@chakra-ui/react`)
- **Styling Engine:** Emotion (`@emotion/react`) & Chakra's System
- **Icons:** React Icons
- **Fonts:** 
  - `Geist` (Sans & Mono) - Next.js default
  - `Space Grotesk` - Headings/Body
  - `IBM Plex Serif` - Accent/Serif text

## Directory Structure
- `src/app/`: Contains the App Router pages and layouts.
  - `page.js`: The main landing page.
  - `layout.js`: Root layout, including font configurations and the Chakra `Provider`.
  - `globals.css`: Global CSS resets and variables.
- `src/components/ui/`: Contains Chakra UI snippets and the main `provider.jsx` wrapper.
- `public/`: Static assets (SVGs, images).

## Building and Running

### Prerequisites
- Node.js (v18+ recommended)
- npm, yarn, pnpm, or bun

### Key Commands

| Command | Description |
| :--- | :--- |
| `npm run dev` | Starts the development server at `http://localhost:3000`. |
| `npm run build` | Builds the application for production. |
| `npm run start` | Starts the production server (after building). |
| `npm run lint` | Runs ESLint to check for code quality issues. |

## Development Conventions

### Styling
- **Primary Method:** Use Chakra UI components (`Box`, `Flex`, `Text`, `Button`, etc.) for layout and styling.
- **Responsiveness:** Use Chakra's responsive array/object syntax (e.g., `fontSize={{ base: "3xl", md: "6xl" }}`).
- **Colors:** The project currently uses hardcoded hex values (e.g., `#545252` for background) but supports Chakra's theme system.
- **Fonts:** Fonts are loaded via `next/font/google` in `src/app/layout.js` and applied via CSS variables (e.g., `var(--font-space-grotesk)`).

### Component Structure
- New components should generally be placed in `src/components`.
- Reusable UI snippets (generated by Chakra CLI) reside in `src/components/ui`.

### State Management
- Uses standard React hooks (`useState`, `useEffect`) and Next.js server/client component patterns.
